Guide — Comprendre les options de proxy
StremThru — à quoi ça sert ?

StremThru est un intermédiaire qui fait transiter les appels API debrid via une IP tierce plutôt que celle de votre serveur. Par défaut, Stream-Fusion utilise une instance publique partagée (stremthru.13377001.xyz). Dans ce mode, l'application ne proxifie pas les liens de streaming — le flux vidéo va directement du CDN du débrider à votre client Stremio. StremThru n'intervient que pour les vérifications de cache et la génération du lien.

Si vous hébergez votre propre instance StremThru, vous pouvez configurer les variables d'environnement correspondantes pour proxifier les liens CDN via votre propre infrastructure.

⚠️ Note cache : Quand StremThru est activé, les vérifications de cache n'utilisent plus l'API native de votre débrider — elles passent uniquement par le cache communautaire de l'instance. Des torrents présents dans votre compte personnel peuvent donc ne pas être détectés comme disponibles, ce qui peut réduire le nombre de résultats affichés.

WARP — pourquoi c'est là ?

Stream-Fusion embarque un container Cloudflare WARP qui fait transiter tous les appels API debrid (vérifications de cache, génération de liens) par une IP Cloudflare fixe plutôt que l'IP brute de votre serveur.

À une certaine époque, AllDebrid bloquait les IP de serveurs au niveau de ses appels API. WARP permet de contourner ça en présentant une IP Cloudflare "propre".

Proxification du streaming — pour qui, pourquoi ?

Voici le fonctionnement normal :

Clic sur un résultat → Stream-Fusion génère un lien CDN → Stremio télécharge le flux directement depuis ce CDN

Dans ce cas, c'est l'IP depuis laquelle vous regardez la vidéo qui est vue par le débrider, pas celle du serveur. Si vous avez un compte par personne, il n'y a aucune raison de proxifier le streaming.

Le problème survient quand plusieurs personnes utilisent le même compte : le débrider voit plusieurs IP différentes accéder au même fichier en simultané, ce qui lève une alerte de partage de compte.

Pour pallier ça, Stream-Fusion propose un mode proxy streaming (proxied_link) : le serveur se place en relai entre vous et le CDN. Le débrider ne voit plus que l'IP du serveur, quelle que soit l'IP des utilisateurs derrière.

L'option playback_proxy permet en plus de faire transiter ce relai à travers WARP. Utile si l'IP de votre serveur elle-même est bloquée (certains VPS Oracle avaient ce problème).

Référence — Variables d'environnement (lecture seule)

Ces variables doivent être définies dans votre fichier .env ou en tant que secrets Docker/Kubernetes. Elles ne peuvent pas être modifiées depuis l'interface admin car elles contiennent des secrets ou des paramètres d'infrastructure.

Sécurité
SESSION_KEY Clé secrète utilisée par Starlette pour signer les cookies de session admin. Doit être une chaîne hexadécimale longue et unique. Si absente, la valeur par défaut non sécurisée est utilisée — un avertissement s'affiche au démarrage. Exemple : openssl rand -hex 32
SECRET_API_KEY Clé secrète utilisée pour l'authentification admin de l'API REST (/api/admin/*). Si absente, ces endpoints sont inaccessibles. Exemple : une chaîne aléatoire de 32+ caractères
CONFIG_SECRET_KEY Clé Fernet (32 bytes base64url) utilisée pour chiffrer les tokens de configuration embarqués dans les URLs d'addon Stremio. Si absente, les tokens ne sont qu'encodés en Base64 (réversible). Ne jamais changer après mise en production — cela invalide toutes les URLs existantes. Générer : python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
Services Debrid
RD_TOKEN Token d'API Real-Debrid. Active automatiquement proxied_link et rd_unique_account.
AD_TOKEN Token d'API AllDebrid. Active automatiquement proxied_link et ad_unique_account.
TB_TOKEN Token d'API TorBox. Active tb_unique_account.
PM_TOKEN Token d'API Premiumize. Active pm_unique_account.
DL_TOKEN Token d'API Debrid-Link. Active dl_unique_account.
ED_TOKEN Token d'API EasyDebrid. Active ed_unique_account.
OC_CREDENTIALS Credentials Offcloud au format email:password. Active oc_unique_account.
PP_CREDENTIALS Credentials PikPak au format email:password. Active pp_unique_account.
Proxy & Réseau
PROXY_URL URL du proxy sortant (SOCKS5 ou HTTP) utilisé pour les appels API debrid et optionnellement pour le streaming. Active automatiquement ad_use_proxy. Exemples : socks5://user:pass@host:1080  |  http://proxy:8080
Base de données PostgreSQL
PG_HOST Nom d'hôte ou IP du serveur PostgreSQL. En Docker Compose : nom du service. Défaut : stremio-postgres
PG_PORT Port TCP de PostgreSQL. Défaut : 5432
PG_USER Nom d'utilisateur PostgreSQL. Défaut : streamfusion
PG_PASS Mot de passe PostgreSQL. Doit correspondre à l'utilisateur PG_USER.
PG_BASE Nom de la base de données PostgreSQL à utiliser. Défaut : streamfusion
Redis
REDIS_HOST Nom d'hôte du serveur Redis. Défaut : redis
REDIS_PORT Port TCP de Redis. Défaut : 6379
REDIS_PASSWORD Mot de passe Redis (optionnel). Laisser vide si Redis n'a pas d'authentification.
TMDB
TMDB_API_KEY Clé d'API The Movie Database (v3). Obligatoire — sans elle, la résolution des métadonnées films/séries est non fonctionnelle. Obtenir sur themoviedb.org/settings/api
